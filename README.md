# منصة إجراءات التحكيم

موقع تدريبي عربي تفاعلي بنظام RTL حول المحاضرة الأولى: **إجراءات التحكيم – من تحريك الخصومة إلى ضبط مسارها**.

## المزايا

- تصميم احترافي ومتجاوب مع الهاتف والحاسوب.
- تسعة محاور تدريبية تغطي بدء الإجراءات، طلب التحكيم، الأطراف، الرد، الدفوع، المواعيد، والجلسة الإجرائية الأولى.
- سيناريو تطبيقي ومحاكاة لقضية مقاولات.
- اختبار ختامي من 10 أسئلة مع حفظ أفضل نتيجة.
- حفظ تقدم المتدرب والتكليف محليًا عبر `localStorage`.
- حاسبة تعليمية لمهلة الرد.
- منشئ مبسط لبنود الأمر الإجرائي الأول.
- وضع ليلي، بحث داخلي، وطباعة ملخص المحاضرة وشهادة الإتمام.
- لا يحتاج إلى خادم أو قاعدة بيانات أو مكتبات خارجية.

## تشغيل الموقع محليًا

افتح ملف `index.html` مباشرة في المتصفح، أو شغّل خادمًا محليًا:

```bash
python -m http.server 8000
```

ثم افتح:

```text
http://localhost:8000
```

## النشر على GitHub Pages

1. أنشئ مستودعًا جديدًا، مثل:
   `arbitration-procedures-training`
2. ارفع جميع الملفات الموجودة في هذا المجلد إلى الفرع `main`.
3. افتح المستودع ثم:
   `Settings` → `Pages`
4. تحت `Build and deployment` اختر:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/ (root)`
5. اضغط `Save`.
6. سيظهر رابط الموقع عادة بالصورة:
   `https://USERNAME.github.io/arbitration-procedures-training/`

## تخصيص سريع

- اسم المحاضر موجود في `index.html`.
- ألوان المنصة موجودة في بداية ملف `styles.css` داخل `:root`.
- أسئلة الاختبار موجودة في مصفوفة `questions` داخل `app.js`.
- عناوين المحاور ومحتواها موجودة داخل أقسام `<section>` في `index.html`.

## تنبيه قانوني

المحتوى تدريبي عام، ولا يُعد رأيًا قانونيًا في نزاع محدد. يجب دائمًا الرجوع إلى اتفاق التحكيم، والقواعد المؤسسية، وقانون مقر التحكيم، والقانون الواجب التطبيق على المسائل محل البحث.

## الترخيص

يمكن استخدام المشروع وتعديله للأغراض التعليمية والمهنية مع الإشارة إلى صاحب المحتوى العلمي.
